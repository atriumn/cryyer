## Issue #10: Build send-on-close workflow

Create `.github/workflows/send-update.yml` — triggers when a draft issue is closed, sends the email.

### Requirements
- Trigger: `issues` event, `closed` action
- Filter: only issues with `draft` label
- Steps:
  1. Parse product name from issue labels
  2. Load product config from `products/{product}.yaml`
  3. Extract email subject + body from issue body
  4. Fetch subscriber list from Supabase via `subscribers.ts`
  5. Send via `send.ts` (Resend)
  6. Add `sent` label to the issue
  7. Post a comment with delivery stats (X emails sent, any failures)

### Edge cases
- Issue closed without `draft` label → skip
- No subscribers for product → log warning, add comment, don't error
- Resend failure → re-open issue, add error comment

### Dependencies
- send.ts (#6)
- subscribers.ts (#7)
- Email template (#8)
