subject: "Cryyer v0.7.0: Social Automation, Gist Storage, and Scoped Package Migration"
---

Cryyer v0.7.0 is now live. This release marks a significant expansion of the toolset, moving beyond email to include a full social media content pipeline and more flexible subscriber storage options.

## Social Content Pipeline
We've introduced a new suite of social CLI commands and providers to automate cross-platform distribution:
- **Social Commands**: New `seed`, `draft`, and `send` commands allow for automated content generation and publishing (#149).
- **Buffer Integration**: Added Buffer as a provider for shipping social drafts (#149).
- **Configuration**: Extended the `Product` type to include social config and support for external configuration directories (#145, #150).
- **Generator**: New social content prompts and draft file I/O for managing multi-platform posts (#148).

## Subscriber Management & Storage
- **GitHub Gist Store**: A new lightweight subscriber backend using GitHub Gists (#114). Note: This requires a classic PAT for write access (#126).
- **Non-interactive Init**: `cryyer init` now supports non-interactive flags for better CI/CD integration and scripting (#106).

## Core Updates & Breaking Changes
- **Scoped Package**: The npm package has been renamed to `@atriumn/cryyer`. Update your `package.json` to track the new scope (#97).
- **Workflow Selection**: Use the new `--pipeline` flag to toggle between `weekly`, `release`, or `both` workflows (#108).
- **LLM Reliability**: Improved parsing logic with a retry mechanism for unparseable LLM responses (#121).

Check the [release notes](https://github.com/atriumn/cryyer/releases/tag/cryyer-v0.6.3) for the full diff and migration details.
