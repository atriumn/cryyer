## Cryyer v0.5.0

This release focuses on automation and workflow flexibility, following a rapid series of updates from v0.1.14 through v0.4.1. 

### Breaking Changes
- **Scoped Package Rename:** The npm package has been renamed to `@atriumn/cryyer`. Update your `package.json` and imports to reflect the new scope (PR #97).
- **Release Tagging:** Release workflows now use specific tag filters for the scoped package to ensure clean versioning (PR #103).

### New Features
- **Non-interactive Init:** You can now run `cryyer init` in CI/CD environments without manual prompts (PR #106).
- **Pipeline Selection:** Added the `--pipeline` flag, allowing you to choose between `weekly`, `release`, or `both` workflow configurations (PR #108).

### Workflow Improvements
- **CI Reliability:** Fixed npm authentication in the release workflow (PR #110) and ensured the `send-email` and `draft-email` workflows build from source to guarantee version accuracy (PR #100, #101).
- **Badge Rendering:** Migrated to shields.io for repository badges to ensure consistent rendering across GitHub (PR #113).
- **Documentation:** Conducted a sweep of the repository to fix stale documentation and align it with the current API (PR #96).

### Version History Recap
- **v0.4.x:** Focused on npm auth and logo rendering fixes.
- **v0.3.x:** Introduced non-interactive initialization.
- **v0.2.x:** Implemented the scoped package migration and CI pipeline flags.

For the full list of changes, view the [commit history](https://github.com/atriumn/cryyer/commits/main).
