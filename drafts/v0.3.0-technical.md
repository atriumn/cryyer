## Cryyer v0.3.0 is here

This release marks a significant milestone in our package management and CI/CD maturity. We've standardized our distribution and streamlined the CLI for automation.

### Breaking Changes & Renaming
- **Scoped Package:** The npm package has been officially renamed to `@atriumn/cryyer` (#97). You must update your `package.json` dependencies to the new scope to receive future updates.
- **Documentation:** We have completed a sweep of the repository to fix stale documentation and align it with the latest API surfaces (#96).

### New Features
- **Non-interactive cryyer init:** The `init` command now supports non-interactive execution (#106). This is a critical addition for developers automating project scaffolding within CI pipelines or custom setup scripts.

### CI/CD and Workflow Improvements
- **Release Workflow:** Fixed tag filters for scoped packages (#103) and refined version parsing in the `send-email` workflow (#101).
- **Build Reliability:** The `draft-email` and `send-email` workflows now build directly from source (#100, #101), ensuring that generated communications accurately reflect the current build state.
- **Release-Please Configuration:** Added support for chore and docs releases to trigger automated versioning (#98).
