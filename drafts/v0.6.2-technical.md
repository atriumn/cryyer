## Release v0.6.2

Cryyer v0.6.2 is now available, focusing on expanding data persistence options and refining CI/CD automation for developer workflows.

### New Features
- **GitHub Gist Subscriber Store**: You can now use a GitHub Gist as your source of truth for subscriber lists (PR #114). This is ideal for lightweight deployments without a dedicated database. Note that this requires a GitHub Personal Access Token (PAT) rather than the default `GITHUB_TOKEN` (PR #119).
- **Pipeline Flexibility**: Added the `--pipeline` flag to allow selection between `weekly`, `release`, or `both` workflow choices (PR #108).
- **Non-Interactive Init**: The `cryyer init` command now supports non-interactive execution (PR #106), simplifying headless setup in CI environments.

### Reliability and Fixes
- **LLM Robustness**: Added automatic retry logic for LLM calls that return unparseable responses (PR #121).
- **CI Workflow Optimizations**: Fixed several issues regarding draft-email triggers, ensuring they only run on PR opens (PR #117) and skip execution if drafts already exist (PR #116).
- **NPM Scoping**: The package has been officially moved to the `@atriumn/cryyer` scope (PR #97). Update your dependencies accordingly.

### Documentation Updates
- Added comprehensive guides for the Gist subscriber store to the README (PR #122).
- Corrected stale documentation regarding configuration schemas (PR #96).

For a full list of changes and PR links, visit the [v0.6.2 release page](https://github.com/atriumn/cryyer/releases).
