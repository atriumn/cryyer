## Cryyer v0.6.0

We have released v0.6.0 with a focus on automation, improved CI/CD workflows, and new storage backends for subscriber management.

### New Features & Enhancements
- **GitHub Gist Subscriber Store (#114):** You can now use a GitHub Gist as your subscriber source of truth, enabling serverless subscriber management without external databases.
- **Non-interactive `cryyer init` (#106):** Added support for non-interactive initialization, making it easier to bootstrap Cryyer in automated environments or scripts.
- **Pipeline Selection (#108):** Introduced the `--pipeline` flag to give developers explicit control over workflow choice (`weekly`, `release`, or `both`).
- **CI/CD Reliability:** Fixed several issues in the `draft-email` logic, including PAT support for CI retriggering (#118) and skipping redundant drafts (#116).

### Breaking Changes & Chores
- **Scoped Package Name (#97):** The npm package has been renamed to `@atriumn/cryyer`. Update your `package.json` to continue receiving updates.
- **Registry Auth (#110):** Improved the release workflow to include proper npm authentication for scoped package publishing.
- **Stale Docs (#96):** Audited and fixed documentation across the repository to match the current API surface.

### Migration Notes
If you are upgrading from a version prior to v0.2.0, ensure you update your imports to reflect the `@atriumn/cryyer` scope. For those running Cryyer in GitHub Actions, the new `draft-email` improvements now build from source to ensure version consistency (#100, #101).

View the full release history on [GitHub](https://github.com/atriumn/cryyer/releases).
