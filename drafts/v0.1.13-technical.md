### Release Update: v0.1.13

We've just shipped **v0.1.13**, focusing on hardening our release pipeline and improving how we target distribution segments.

#### Key Changes

*   **Multi-audience Pipeline Integration ([PR #93](https://github.com/atriumn/cryyer/pull/93))**: We’ve officially wired multi-audience support into the release pipeline. This allows you to define and target specific user segments directly during the ship process. If you’re building custom distribution hooks, you can now reference these audience identifiers within the deployment lifecycle.
*   **Release Automation ([PR #91](https://github.com/atriumn/cryyer/pull/91))**: Finalized the automation chores for the v0.1.12 baseline to ensure smoother version increments moving forward.

#### What this means for you

Beta testers can now start testing segmented rollouts. By integrating audiences into the pipeline, we're moving away from monolithic releases toward more granular, environment-specific distribution. Check the updated API docs for the new audience parameters in the release schema.
