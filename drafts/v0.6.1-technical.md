## v0.6.1 Release

Cryyer v0.6.1 is now live, focusing on expanding storage options and improving the reliability of our automated workflows.

### GitHub Gist Subscriber Store
We’ve introduced a new subscriber backend using GitHub Gists (#114). This allows for a lightweight, serverless way to manage your mailing list. Please note that using the Gist store requires a Personal Access Token (PAT) for write access, as the default `GITHUB_TOKEN` is insufficient for this implementation (#119).

### CLI & Pipeline Enhancements
- **Pipeline Control**: The new `--pipeline` flag (#108) allows you to toggle between `weekly`, `release`, or `both` workflows, giving you granular control over when emails are generated.
- **Resilient LLM Calls**: Added logic to retry LLM calls once if the initial response is unparseable (#121), reducing CI failures due to transient formatting issues.
- **Non-interactive Init**: `cryyer init` now supports non-interactive execution (#106), streamlining setup for automated environments.

### Breaking Changes & Internal Updates
- **Namespace Change**: The npm package has been renamed to `@atriumn/cryyer` (#97). Please update your dependencies accordingly.
- **Workflow Optimization**: The `draft-email` action now skips execution if drafts already exist (#116) and specifically requires a PAT for pushes to ensure CI re-triggers correctly (#118).

Full release notes and PR history can be found on [GitHub](https://github.com/atriumn/cryyer/releases).
