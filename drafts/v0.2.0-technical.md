## Cryyer v0.2.0

This release focuses on infrastructure stabilization and a significant update to our package distribution. Most notably, we have migrated to a scoped NPM package and resolved critical issues in our CI/CD pipelines.

### Breaking Change: NPM Scoped Package
In PR #97, we officially renamed the npm package to **@atriumn/cryyer**. Developers building on Cryyer will need to update their dependencies and import statements to reflect this change.

- Update your package.json: `npm install @atriumn/cryyer`

### Workflow and CI Improvements
We’ve implemented several fixes to ensure the stability of the build process and the accuracy of our release cycle:

- **Build from Source:** Fixed a bug in the `draft-email` workflow that prevented successful builds from source (#100).
- **Release Automation:** Reconfigured `release-please` settings to better handle documentation and chore-related releases, ensuring cleaner versioning for the project (#98).
- **Documentation Refresh:** Conducted a sweep of the repository to fix stale documentation, ensuring that the README and API references match the current state of the codebase (#96).
