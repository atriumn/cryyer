## Cryyer v0.2.2

This release focuses on improving package distribution and stabilizing our release pipelines. 

### Scoped Package Migration
The most significant change in this cycle is the renaming of our npm package to `@atriumn/cryyer` (PR #97). This move to a scoped package allows us to better manage the project under the Atriumn organization. If you are building on top of Cryyer, update your dependencies to `@atriumn/cryyer` to receive future updates.

### CI/CD and Workflow Improvements
We have hardened our automation to support the new scoped structure and improve release reliability:
- **Tag Filtering:** Fixed the release workflow tag filter to correctly identify scoped package releases (PR #103).
- **Workflow Fixes:** Addressed version parsing and source build issues within the `send-email` and `draft-email` workflows (PR #101, #100).
- **Release Automation:** Configured `release-please` to handle `docs` and `chore` commits, ensuring consistent versioning even for non-functional changes (PR #98).

### Documentation and Maintenance
- **Stale Docs:** Performed a comprehensive cleanup of stale documentation across the repository to ensure current API usage is correctly reflected (PR #96).
- **Recent Releases:** This update follows the rapid iteration of v0.2.1 and v0.2.0, which finalized the transition to the Atriumn namespace.

For the full diff and specific commit logs, view the [v0.2.2 release notes](https://github.com/atriumn/cryyer/releases).
