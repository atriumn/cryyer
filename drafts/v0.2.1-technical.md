## Cryyer v0.2.1 Release

We have just pushed v0.2.1. This update focuses on internal infrastructure stability and a significant change to our distribution channel.

### Breaking Change: NPM Scoping
The most impactful change in this release cycle is the migration to a scoped NPM package. This ensures better ownership and security for the library:
- Renamed package to `@atriumn/cryyer` (PR #97).

If you are using Cryyer as a dependency, please update your `package.json` and change your imports to `@atriumn/cryyer` to continue receiving updates.

### Workflow Improvements
We have addressed several issues in our automation pipelines to streamline the release process:
- Fixed `send-email` and `draft-email` workflows to build from source and correctly parse versions (PR #100, PR #101).
- Updated `release-please` configuration to properly trigger on documentation updates and chore commits (PR #98).

### Documentation & Chores
- Purged stale documentation across the repository to reflect current API usage (PR #96).
- Standardized release automation for smoother future updates.

View the full release notes and tags on [GitHub](https://github.com/atriumn/cryyer/releases).

Thanks for contributing to and building on Cryyer.
