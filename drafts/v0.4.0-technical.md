## Cryyer v0.4.0 is live

This release focuses on streamlining automation and refining the CLI experience for CI/CD environments. We've moved toward better support for headless environments and more granular control over notification pipelines.

### Key Features
- **Pipeline Workflow Control**: PR #108 introduces the `--pipeline` flag. You can now explicitly choose between `weekly`, `release`, or `both` workflow choices, providing better control for teams with custom release cadences.
- **Non-interactive Init**: With PR #106, `cryyer init` now supports a non-interactive mode. This is essential for automated setup scripts and containerized environments where TTY input isn't available.
- **NPM Package Renaming**: As of PR #97, the package has been moved to a scoped registry: `@atriumn/cryyer`. Update your dependencies to point to the new scope to receive future updates.

### Internal Improvements and Fixes
- **Workflow Stability**: We fixed issues in the `send-email` and `draft-email` workflows where version parsing failed; the system now builds from source reliably (PR #100, PR #101).
- **Release Automation**: Updated `release-please` configurations to properly filter tags for scoped packages and include `docs`/`chore` changes in release calculations (PR #98, PR #103).
- **Documentation**: Stale documentation has been purged across the repository to align with the current v0.4.0 API (PR #96).

For a full list of changes and the raw diff, visit the release page.
